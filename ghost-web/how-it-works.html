Component 01

Ghost Core

A Bitcoin Core fork that stays consensus-compatible. Ghost Core validates the same blocks, follows the same rules, and speaks the same protocol as every other Bitcoin node.

The difference? You control your node's policy. Choose what transactions to relay, what data to store, and how to build blocks.

  • 100% Bitcoin consensus compatible
  • Custom mempool and relay policies
  • BUDS data classification system
  • Ghost Mode for private operation
  • Flexible pruning (VW/OW/AW windows)

Component 02

Ghost Pool

Every Ghost node is its own mining pool.

No centralized pool operator. No middleman taking fees. No censorship risk. When a miner connected to your node finds a block, you keep 100% of the transaction fees.

For miners, it looks like any other pool — same Stratum protocol, same familiar setup. The difference is in who controls the block template: you do.

  • Standard Stratum V1/V2 protocol
  • 1% fee on block subsidy only
  • 100% of TX fees go to winning node
  • Decentralized share consensus
  • No censorship possible

Component 03

Ghost Pay

Fast, private payments that settle to Bitcoin. No channels. No liquidity management. No routing failures.

Lock Bitcoin once on L1. This creates your spending capacity.

Send instant private payments. Confirms in ~10 seconds.

Settle back to Bitcoin anytime. Your keys, your coins.
Not Lightning. Simpler.
Lightning requires opening channels, finding routes, and managing liquidity. Ghost Pay works more like cash — lock it once, spend freely, settle when ready. No channel management, no routing failures, no inbound liquidity problems.

Amounts hidden with ZK proofs. No one can see how much you're sending.
📴
Offline Receive
Receive payments even when offline. Claim them when you reconnect.
