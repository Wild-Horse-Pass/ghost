Fire Ping Load Balancer

Automatic latency-based routing to the nearest Ghost Pool node. One endpoint, global coverage.

How Fire Ping Works

The ghost-coordinator service routes miners to the lowest-latency pool node automatically.

1
Miner Connects
Miner connects to pool.bitcoinghost.org:3333 (SV1) or :34255 (SV2)
2
Coordinator Measures Latency
The ghost-coordinator pings all available pool nodes and measures round-trip time from the miner's location.
3
Route to Nearest Node
Miner is transparently proxied to the pool node with the lowest latency. No configuration needed.
✓
Mining Begins
Connection is maintained to the selected node. If node goes offline, automatic failover to next-best node.

Stratum Endpoints

Use these endpoints for automatic load balancing across all Ghost Pool nodes.

Recommended

Stratum V1 (SV1)

stratum+tcp://pool.bitcoinghost.org:3333

Most compatible. Works with all ASIC miners including Antminer, Whatsminer, Avalon, and software miners like CGMiner/BFGMiner.

Fire Ping Enabled Port 3333
High Performance

Stratum V2 (SV2)

stratum+tcp://pool.bitcoinghost.org:34255

Next-generation protocol with encrypted connections, reduced bandwidth, and job negotiation. Requires SV2-compatible firmware.

Fire Ping Enabled Port 34255
Translator

SV1 → SV2 Translation

Automatic via ghost-translator

SV1 miners connecting to Ghost Pool are automatically translated to SV2 for communication with pool nodes. No miner-side changes needed.

Why Fire Ping?

Benefits of latency-based routing for miners.

⚡
Lower Latency
Every millisecond counts. Lower latency means less stale shares and higher effective hashrate.
🔄
Automatic Failover
If your connected node goes offline, Fire Ping automatically routes you to the next-best node.
🌍
Global Coverage
One endpoint works worldwide. No need to manually select regional servers.
📊
Health Monitoring
Continuous health checks ensure miners are never routed to degraded nodes.
🔧
Zero Configuration
Just point your miner to the Fire Ping endpoint. Everything else is automatic.
🔐
SV2 Translation
Legacy SV1 miners get automatic SV2 translation for improved efficiency.

For Node Operators

Running the ghost-coordinator service to join the Fire Ping network.

# The ghost-coordinator runs alongside ghost-pool
# It registers your node with the Fire Ping network

# Start the coordinator service
ghost-coordinator --config /etc/ghost/coordinator.toml

# Or via systemd
sudo systemctl start ghost-coordinator

# Configuration (coordinator.toml)
[coordinator]
listen_addr = "0.0.0.0:8334"
stratum_v1_port = 3333
stratum_v2_port = 34255
announce_endpoint = "pool.bitcoinghost.org"
region = "us-west"
Earn +3 Shares for Public Mining

Node operators who enable public mining and join the Fire Ping network earn an additional +3 shares in the reward system. Your node serves external miners and strengthens the decentralized pool infrastructure.
